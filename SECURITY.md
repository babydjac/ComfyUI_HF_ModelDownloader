# Security Policy

## Supported versions

Security fixes are generally applied to the latest state of the default branch.

## Reporting a vulnerability

- Do not post exploit details, tokens, or private environment information in a public issue.
- Use GitHub's private vulnerability reporting for this repository if it is enabled.
- If private reporting is not available, open a minimal public issue requesting a secure reporting channel without sharing sensitive details.

When reporting a vulnerability, include:

- A short description of the issue
- Impact and affected areas
- Clear reproduction steps or a proof of concept
- Any suggested mitigation if you have one

The goal is to acknowledge reports quickly, investigate responsibly, and ship a fix or mitigation as soon as practical.
